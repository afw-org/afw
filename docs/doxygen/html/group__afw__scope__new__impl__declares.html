Detailed Description

This file should only be included in c source files that implement interface afw_scope_new.

You can find a skeleton for using this #include file in the corresponding source directory in file generated/interface_closet/skeleton_afw_scope_new.c.

Mutiple *_impl.declares.h can be included in each c source file, but only one for each interface name.

After the #include for afw_scope_new_impl_declares.h:

  1. declarations will exist for all of the interfaces methods with names of impl_afw_scope_new_ followed by the method name.
  2. rti will be defined with a name of impl_afw_scope_new_rti.
  3. inf will be defined with a name of impl_afw_scope_new_inf.

Before including, #define the following symbols:

  • AFW_IMPLEMENTATION_ID - Implementation id string for this implementation.
  • AFW_IMPLEMENTATION_INF_SPECIFIER - (optional) defaults to static.
  • AFW_IMPLEMENTATION_INF_LABEL - (optional) defaults to impl_afw_scope_new_inf.

Example:

static const sometype_t impl_myimpl_stuff = {
...
};
#define AFW_IMPLEMENTATION_ID "myimpl"
#define AFW_IMPLEMENTATION_SPECIFIC &impl_myimpl_stuff
#include "afw_scope_new_impl_declares.h"
A second #include "afw_scope_new_impl_declares.h" can be specified if
" * #define "AFW_SCOPE_NEW_INF_ONLY" is specified to generate inf only."
