24 """Command afwdev is a development tool used to create, extend, maintain, and

30 """,
31  "epilog":
32  """
33 Directory structure of an AFW package
34 -------------------------------------
35 
36 The basic structure of an AFW package is:
37 
38 <AFW package root directory>
39 |-- doc/ # Generated docs
40 |-- generated/ # Produced by generate subcommand
41 | |-- *.cmake # Generated cmake files
42 |-- src/ # Commonly src/ contains source directories
43 | |-- <SRCDIR>/ # One or more command, extension, or server
44 | |-- generate/ # Processed by generate subcommand
45 | |-- generated/ # Produced by generate subcommand
46 | | |-- c source # Generated *.h and .c files
47 | | |-- *.cmake # Generated cmake files
48 | | |-- ... # Other generated files and directories
49 | |-- c source # <SRCDIR>_*.h and .c files
50 | |-- CMakeLists.txt # CMakeLists.txt for <SRCDIR>
51 | |-- ... # Other source files and directories
52 |-- .editorconfig # Source editor coding style
53 |-- .gitattributes # Command git attributes for pathnames
54 |-- .gitignore # Command git files to ignore
55 |-- afw-package.json # AFW package file managed by the afwdev
56 |-- CMakeLists.txt # Root CMakeLists.txt for afw package
57 |-- COPYING # Licensing information.
58 |-- Doxyfile # Doxygen configuration file
59 |-- LICENSE # The license.
60 |-- package.json # Used to build a JavaScript package
61 |-- <PACKAGE_ID>_git_info.sh # Used to produce <PACKAGE_ID>_git_info.h
62 |-- <PACKAGE_ID>_git_info.h # Header file containing git information
63 |-- README.md # Repository README.md
64 |-- miscellaneous directories # Other directories
65 |-- miscellaneous files # Other files
66 
67 Zero or more <SRCDIR> directories can exist. Each <SRCDIR> contains the
68 source for one command, extension, or server implementation provided by the
69 AFW package. There can also be <SRCDIR> directories for other buildtypes.
70 This information is kept in afw-package.json.
71 
72 All make-* subcommands make a new directory. The make-afw-package subcommand
73 makes a skeleton package directory that includes an afw-package.json file.
74 The other make-* subcommands make a skeleton directory in the src/ directory
75 and adds an entry to the afw-package.json "srcdir" property.
76 
77 All add-* subcommands expect an existing "srcdir" entry in the afw-package.json
78 file that is either their implementation id or the value specified with the
79 --srcdir argument.
80 
81 All add-* and make-* subcommands produces files with @todo comments that
82 should be searched for and replaced as appropriate.
83 
84 If certain optional arguments are not specified, a prompt will occur that
85 allows the default value to be overridden. If the --noprompt argument is
86 specified or the specific optional argument is defined to never prompt, this
87 prompting will not occur. Specify the --prompt argument to insure prompting
88 occurs for all missing optional parameters.
89 
90 Examples
91 --------
92 
93 1) Make an AFW package "mypackage" in directory ~/mypackage:
94 
95  afwdev make-afw-package mypackage ~/mypackage
96 
97 2) Make an extension "myextension" srcdir in "mypackage":
98 
99  cd ~/mypackage
100  afwdev make-extension myextension
101 
102 3) Add an adaptor type with id "myadaptor" to "myextension":
103 
104  cd ~/mypackage
105  afwdev add-adaptor-type myadaptor --srcdir myextension
106 
107 4) Make command "mycommand" srcdir in "mypackage":
108 
109  cd ~/mypackage
110  afwdev make-command mycommand
111 
112 5) Generate afwdev maintained files for "mypackage" after changes:
113 
114  cd ~/mypackage
115  afwdev generate --srcdir-pattern \*
116 
117 6) Run tests defined in "mypackage":
118 
119  cd ~/mypackage
120  afwdev test --srcdir-pattern \*
121
